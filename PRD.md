# PRD: 감사 전 재무비율 분석 도구 (Pre-Audit Financial Ratio Analyzer)

## 1. 개요

| 항목 | 내용 |
|---|---|
| **What** | DART 전자공시 감사보고서 링크를 입력받아 재무제표를 자동 추출하고, 전전기·전기·당기 3개년의 유동비율/부채비율/자기자본비율/ROA/ROE/PER/PBR 등을 테이블과 그래프로 제시하는 프로그램 |
| **Why** | 본격적인 감사절차 착수 전, 중요왜곡표시위험(RMM)이 발생할 수 있는 비율을 사전에 포착하고, 당기 감사의견 형성의 근거자료로 활용하기 위함 |
| **Who / Where** | 회계법인 소속 회계사 (감사팀) |
| **When** | 본격적인 감사절차 시작 전 (예비조사/위험평가 단계) |
| **How** | 감사보고서 링크 → 재무제표 파싱 → 비율 계산 → 테이블 제시 → 그래프 시각화 |

## 2. 목표 및 성공 기준

- 감사팀이 DART 링크 하나만 입력하면 5분 내에 3개년 재무비율 분석표를 확보할 수 있어야 함
- 계산된 비율이 실제 재무제표 수치와 교차검증했을 때 오차 없이 일치해야 함
- 전기 대비 이상 변동(예: 부채비율 급증, 유동비율 급락)이 시각적으로 즉시 식별 가능해야 함

## 3. 사용자 스토리

- 회계사로서, 나는 DART 감사보고서 링크를 붙여넣기만 하면 회사명과 사업연도를 자동으로 인식하고 싶다.
- 회계사로서, 나는 전전기/전기/당기 비율을 한 테이블에서 비교하고 싶다.
- 회계사로서, 나는 비율의 3개년 추이를 그래프로 보고 이상치를 빠르게 파악하고 싶다.
- 회계사로서, 나는 계정과목 매핑이 실패한 경우 경고를 받아서 수치를 맹신하지 않고 싶다.

## 4. 범위 (Scope)

### 포함
- DART 감사보고서(사업보고서 첨부 감사보고서) 뷰어 링크 파싱
- 단일회사 3개년 재무비율 계산 및 표/그래프 제시
- 계정과목 매핑 실패 시 경고 표시

### 제외 (1차 버전 기준)
- 여러 회사 동시 비교 (추후 확장)
- 연결/별도 재무제표 동시 비교 (1차는 하나만 선택)
- 감사의견 자동 판단 로직 (수치 제시까지만, 판단은 회계사 몫)

## 5. 단계별 구현 로드맵

### Phase 1 — 링크 파싱 및 API 연동
**목표**: DART 감사보고서 링크에서 필요한 식별자를 추출하고, 재무제표 원본 데이터를 가져온다.

- DART 뷰어 URL(예: `dart.fss.or.kr/dsaf001/main.do?rcpNo=...`)에서 `rcept_no`(접수번호) 파싱
- `rcept_no`로 공시서류 원문 조회 API 또는 고유번호 API를 통해 `corp_code`, `bsns_year`, `reprt_code` 역산
- `fnlttSinglAcntAll`(단일회사 전체 재무제표) API 호출 → 당기/전기/전전기 원본 계정 데이터 확보
- **Definition of Done**: 링크 하나를 입력하면 회사명, 사업연도, 3개년 원본 재무제표 데이터프레임이 출력됨

**주의사항**
- 감사보고서 자체는 별도 첨부문서(XML/PDF)로 존재하고, 재무수치는 사업보고서 재무제표 API에서 가져오는 구조이므로 "감사보고서 링크 → 사업보고서 재무제표"로 연결하는 매핑 로직이 필요함
- rcept_no만으로 corp_code를 바로 못 얻는 경우, 공시서류 원문 API(document.xml)를 통한 역조회 단계가 하나 더 필요할 수 있음 → 이 부분은 실제 링크로 먼저 테스트 필요

### Phase 2 — 재무비율 계산 및 테이블 제시
**목표**: 원본 계정 데이터를 표준화하고 핵심 재무비율을 3개년치 계산한다.

계산 대상 비율:
- 유동비율 = 유동자산 / 유동부채
- 부채비율 = 부채총계 / 자본총계
- 자기자본비율 = 자본총계 / 자산총계
- ROA = 당기순이익 / 자산총계(평균)
- ROE = 당기순이익 / 자본총계(평균)
- 매출총이익률, 영업이익률, 순이익률 (감사 위험평가에 유용하여 추가 권장)

- 계정과목명 표준화 딕셔너리 구축 (회사별 상이한 명칭 대응: "매출액"/"영업수익" 등)
- 매핑 실패 계정은 화면에 경고로 표시, 수치는 공란 처리 (임의 추정 금지)
- 결과를 "연도 / 비율명 / 값" 롱포맷 데이터프레임으로 반환 → 추후 확장 대비
- 테이블 UI: 전전기·전기·당기를 컬럼으로, 비율명을 행으로 배치, 전기 대비 증감률 컬럼 추가
- **Definition of Done**: 회사 하나에 대해 3개년 비율 표가 정확히 출력되고 CSV 다운로드 가능

### Phase 3 — 그래프 시각화
**목표**: 비율별 3개년 추이를 시각적으로 비교하고 이상 변동을 강조한다.

- 비율별 라인차트/바차트 (plotly)
- 전기 대비 임계치(예: ±20% 이상 변동) 초과 시 강조 표시 (색상 변경 또는 주석)
- 유동성/레버리지/수익성 지표를 탭 또는 섹션으로 그룹화하여 배치
- **Definition of Done**: 테이블에서 확인한 이상 변동이 그래프에서도 한눈에 식별 가능

### Phase 4 — PER/PBR 확장 (별도 데이터 소스 필요)
**목표**: 시장가치 기반 비율을 추가한다.

- ⚠️ DART API는 재무제표만 제공하고 주가 데이터는 제공하지 않음 → 별도 소스 필요
- 상장사 주가/시가총액: 네이버금융, KRX 정보데이터시스템, 또는 유료 시세 API 중 선택
- EPS = 당기순이익 / 발행주식수, BPS = 자본총계 / 발행주식수 (발행주식수는 DART 재무제표 주석 또는 별도 API에서 확보)
- PER = 주가 / EPS, PBR = 주가 / BPS
- 비상장사의 경우 PER/PBR 산출 불가 → UI에서 "비상장사는 해당 비율 계산 불가" 안내 필요
- **Definition of Done**: 상장사 기준으로 PER/PBR이 정확히 계산되고, 비상장사는 명확한 안내 문구가 표시됨

### Phase 5 — 배포 및 보안 처리
**목표**: GitHub/Supabase 배포 시 API 키 노출 및 서비스 안정성 문제를 방지한다.

- API 키는 `.env` + `.gitignore` 처리, 배포 플랫폼의 시크릿 매니저 사용
- 사용자가 직접 자신의 DART API 키를 입력하는 방식 유지 (서버 키 노출 최소화)
- 과도한 호출 방지를 위한 캐싱 및 요청 제한 로직
- 면책 문구 추가: "본 도구는 참고용 분석 자료이며 감사의견 형성의 유일한 근거가 될 수 없음"

## 6. 비기능 요구사항

- **정확성**: 계산된 수치는 반드시 원본 공시자료와 대사(reconciliation) 가능해야 함
- **투명성**: 계정 매핑 로직이 임의 추정을 하지 않고, 실패 시 명확히 표시
- **감사 실무 적합성**: 단순 투자지표가 아니라 위험평가 관점에서 유의미한 변동을 강조하는 방향으로 설계

  - ROA/ROE 분모(자산총계/자본총계 평균)는 전기 데이터가 있으면 (당기+전기)/2, 없으면(전전기의 경우) 기말값을 그대로 사용
  - `main.py`에 비율 테이블 출력, `--csv PATH` 옵션으로 CSV 저장(utf-8-sig) 추가, 면책 문구 추가
  - `tests/test_ratios.py` 신규 6건 — account_id/account_nm 폴백 매칭, 매출총이익 파생 계산, 매핑 실패 경고, ROA/ROE 평균 로직, 증감률, 분모 결측 시 공란 처리 검증
- **실데이터 검증**: 삼성전자 2025 사업보고서(위 링크)로 실행 → 매핑 실패 0건, 파생 계산 0건(매출총이익 직접 태깅됨). 유동비율/부채비율/자기자본비율을 원본 재무제표 수치와 수동 대사(reconciliation)하여 오차 없이 일치 확인 (예: 당기 유동비율 139.08% = 유동자산 101.44조 / 유동부채 72.94조)
- **결론**: Phase 2 Definition of Done(3개년 비율 표 출력 + CSV 다운로드) 충족. 테이블 UI는 현재 CLI 텍스트 출력 수준 — Streamlit 등 그래픽 UI 전환은 Phase 3(그래프 시각화) 착수 시 함께 검토 필요 (오픈 이슈로 남김)
- **미검증/오픈 이슈**: 매출총이익 미태깅 + 매출원가도 없는 회사(매출총이익 자체 표시 불가한 업종, 예: 금융업) 케이스는 실데이터로 아직 미검증. 계정과목명이 표준 alias 목록에 없는 회사(중소기업 등 비표준 명칭 사용) 케이스도 미검증
