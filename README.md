# fs-analyze

DART(전자공시시스템) 감사보고서/사업보고서 뷰어 링크 하나만으로 최근 3개년(전전기/전기/당기) 재무제표를 자동으로 가져와 핵심 재무비율을 계산하고, 그래프가 포함된 엑셀 보고서를 생성하는 CLI 도구입니다.

회계법인의 감사 착수 전(pre-audit) 단계에서, 감사인이 링크 하나만 넣으면 재무상태표/손익계산서 주요 항목과 유동비율·부채비율·ROA·ROE 등 8개 재무비율을 자동 산출해 이상 변동(전기 대비 ±20% 이상)을 빠르게 스크리닝할 수 있도록 만들었습니다.

## 주요 기능

- DART 뷰어 링크(`rcpNo`)만으로 회사명/사업연도를 자동 역산 (`src/resolver.py`)
- 3개년 재무제표 전체 계정 조회 (`src/dart_client.py`, `src/financials.py`)
- IFRS 표준계정코드(`account_id`) 우선 + 계정과목명 폴백 매칭으로 회사마다 다른 계정명(예: "매출액"/"영업수익")을 표준화 (`src/ratios.py`)
  - 표준계정 매핑에 실패하면 **임의로 추정하지 않고 공란 + 경고**만 표시합니다.
  - 매칭 대상은 해당 계정이 속한 재무제표 구분(`sj_nm`: 재무상태표/손익계산서 등)으로 먼저 좁힌 뒤, 그 안에서 동일 계정코드로 매칭된 값이 서로 다르면(중복 매칭 충돌) 임의로 첫 값을 택하지 않고 공란 + 경고 처리합니다. `scripts/check_duplicate_accounts.py`로 실제 DART 데이터에서 이런 충돌이 있는지 여러 회사를 순회하며 점검할 수 있습니다.
- 8개 재무비율(유동비율/부채비율/자기자본비율/ROA/ROE/매출총이익률/영업이익률/순이익률) 및 전기대비증감률 계산
- 결과를 그래프 포함 `.xlsx` 파일로 저장 (`src/excel_export.py`)
  - 재무비율 표에서 전기대비 ±20% 이상 급변동 항목 자동 강조(조건부 서식)
  - "재무비율" 시트: 핵심 비율(유동비율/부채비율/자기자본비율/ROA/ROE) 3개년 라인차트 — 감사 위험 판단의 메인 그래프
  - "상세보기_원항목추이" 시트: 유동자산/비유동자산, 유동부채/비유동부채 구성 스택 바 차트 — 비율이 동일해도 절대금액 구조가 바뀐 경우를 보조적으로 확인하는 드릴다운 자료
  - "검증" 시트: 비율 계산 로직과 완전히 별개의 코드로 재계산한 대사 결과(유동비율/부채비율/자기자본비율) + 재무상태표 등식(자산=부채+자본, 자산=유동자산+비유동자산) 확인 결과를 실행할 때마다 자동으로 기록 (`src/verification.py`). 불일치가 있으면 빨간색으로 강조되고 콘솔에도 경고가 출력됨

## 프로젝트 상태

Phase 1(링크 파싱 + DART API 연동)~Phase 3(비율 계산 + 그래프 포함 엑셀 출력)까지 실제 DART 공시(삼성전자, 덴티움 사업보고서)로 검증 완료. 전체 로드맵과 진행 로그는 [`PRD.md`](./PRD.md) 참고.

## API 키 설정

DART Open API 키가 필요합니다 ([opendart.fss.or.kr](https://opendart.fss.or.kr)에서 무료 발급). 프로젝트 루트에 `.env` 파일을 만들고 `.env.example`을 참고해 아래와 같이 작성하세요.

```
DART_API_KEY=발급받은_키
```

`.env`는 `.gitignore`에 포함되어 있어 저장소에 올라가지 않습니다.

## 사용법

```bash
python main.py "https://dart.fss.or.kr/dsaf001/main.do?rcpNo=20260323001196&dcmNo=11166205"

# 별도재무제표 기준으로 강제 조회 (기본값은 아래 자동 판별)
python main.py "<DART 링크>" --fs-div OFS

# 결과 저장 위치 지정
python main.py "<DART 링크>" --output-dir ./output
```

`--fs-div`를 지정하지 않으면 다음 순서로 연결/별도를 자동 판별합니다: (1) 링크에 `dcmNo`가 있으면 그 문서가 "감사보고서"(별도)인지 "연결감사보고서"(연결)인지 확인해 우선 사용 — 사용자가 실제로 클릭해서 가져온 링크의 의도를 최우선으로 반영, (2) dcmNo가 없거나 판별 불가면 연결재무제표(CFS)를 먼저 시도, (3) 그 기준의 재무제표가 실제로 공시되어 있지 않으면 반대쪽으로 자동 전환. 계정과목 내용(비지배지분 유무 등)으로 추론하지 않고 링크·API 응답 자체로만 판별합니다. 실행하면 콘솔에 어떤 근거로 연결/별도가 선택됐는지 안내가 출력되고, `{회사명}_{연결|별도}_결과.xlsx` 파일이 생성됩니다(안내/재무제표/재무비율/상세보기_원항목추이 4개 시트 + 그래프 포함, "안내" 시트에 판단 근거까지 명시).

## API 키 없이 결과 미리 보기
