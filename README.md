# fs-analyze

DART(전자공시시스템) 감사보고서/사업보고서 뷰어 링크 하나만으로 최근 3개년(전전기/전기/당기) 재무제표를 자동으로 가져와 핵심 재무비율을 계산하고, 그래프가 포함된 엑셀 보고서를 생성하는 CLI 도구입니다.

회계법인의 감사 착수 전(pre-audit) 단계에서, 감사인이 링크 하나만 넣으면 재무상태표/손익계산서 주요 항목과 유동비율·부채비율·ROA·ROE 등 8개 재무비율을 자동 산출해 이상 변동(전기 대비 ±20% 이상)을 빠르게 스크리닝할 수 있도록 만들었습니다.

## 주요 기능

- DART 뷰어 링크(`rcpNo`)만으로 회사명/사업연도를 자동 역산 (`src/resolver.py`)
- 3개년 재무제표 전체 계정 조회 (`src/dart_client.py`, `src/financials.py`)
- IFRS 표준계정코드(`account_id`) 우선 + 계정과목명 폴백 매칭으로 회사마다 다른 계정명(예: "매출액"/"영업수익")을 표준화 (`src/ratios.py`)
  - 표준계정 매핑에 실패하면 **임의로 추정하지 않고 공란 + 경고**만 표시합니다.
  - 매칭 대상은 해당 계정이 속한 재무제표 구분(`sj_nm`: 재무상태표/손익계산서 등)으로 먼저 좁힌 뒤, 그 안에서 동일 계정코드로 매칭된 값이 서로 다르면(중복 매칭 충돌) 임의로 첫 값을 택하지 않고 공란 + 경고 처리합니다. `scripts/check_duplicate_accounts.py`로 실제 DART 데이터에서 이런 충돌이 있는지 여러 회사를 순회하며 점검할 수 있습니다.
- 8개 재무비율(유동비율/부채비율/자기자본비율/ROA/ROE/매출총이익률/영업이익률/순이익률) 및 전기대비증감률 계산
- 결과를 그래프 포함 `.xlsx` 파일로 저장 (`src/excel_export.py`)
  - 재무비율 표에서 전기대비 ±20% 이상 급변동 항목 자동 강조(조건부 서식)
  - "재무비율" 시트: 핵심 비율(유동비율/부채비율/자기자본비율/ROA/ROE) 3개년 라인차트 — 감사 위험 판단의 메인 그래프
  - "상세보기_원항목추이" 시트: 유동자산/비유동자산, 유동부채/비유동부채 구성 스택 바 차트 — 비율이 동일해도 절대금액 구조가 바뀐 경우를 보조적으로 확인하는 드릴다운 자료
  - "검증" 시트: 비율 계산 로직과 완전히 별개의 코드로 재계산한 대사 결과(유동비율/부채비율/자기자본비율) + 재무상태표 등식(자산=부채+자본, 자산=유동자산+비유동자산) 확인 결과를 실행할 때마다 자동으로 기록 (`src/verification.py`). 불일치가 있으면 빨간색으로 강조되고 콘솔에도 경고가 출력됨
- 금융업(은행/보험/증권/지주회사) 자동 감지 및 경고 (`src/industry_detect.py`) — 계정과목명(이자수익/보험계약부채/예수부채 등)으로 감지해, 재무제표 구조 차이 때문에 다수 비율이 공란 처리되는 것을 "프로그램 오류"로 오해하지 않도록 콘솔·엑셀 양쪽에 눈에 띄게 경고
- 비율이 공란일 때 원인 계정명으로 사유 설명 — "매출원가가 없어 산출되지 않습니다"처럼 실제로 없는 원천 계정을 짚어서, 콘솔 안내 문구와 엑셀 "재무비율" 시트의 해당 셀 메모(comment)로 함께 표시 (`src/ratios.py`)
- 전기 대비 당기 증감 스크리닝(분석적 절차) — 재무상태표/손익계산서/현금흐름표의 개별 계정(표준화 이전 원본) 전체를 대상으로 증감률 ±10%를 초과하는 계정을 자동 추림 (`src/screening.py`). 신규계정(전기=0)·부호전환(흑자↔적자)은 %를 억지로 계산하지 않고 별도 표시하고, 자산총계 대비 1% 미만인 소액 계정은 노이즈 방지를 위해 제외. "스크리닝_전기당기증감" 시트에 증감액 내림차순으로 정리되고 ±30% 초과/±10~30%가 색상으로 구분되며, 계산 로직과 완전히 별개의 코드로 재계산·전수 검사하는 독립 검증도 기존 "검증" 시트에 함께 기록됨

## 프로젝트 상태

Phase 1(링크 파싱 + DART API 연동)~Phase 3(비율 계산 + 그래프 포함 엑셀 출력)까지 실제 DART 공시(삼성전자, 덴티움 등)로 검증 완료. 이후 실사용 중 발견된 이슈 대응(연결/별도 자동판별, 계정 중복 매칭 방어, 금융업 감지, 비율 공란 사유 설명, 전기 대비 당기 증감 스크리닝 등)도 반영 완료. Phase 4(PER/PBR)는 감사 위험평가 목적에 덜 부합한다고 판단해 진행하지 않기로 결정됨(대신 증감 스크리닝 기능으로 대체). Phase 5(배포/보안)는 아직 미착수. 전체 로드맵과 진행 로그는 [`PRD.md`](./PRD.md) 참고.

## API 키 설정

DART Open API 키가 필요합니다 ([opendart.fss.or.kr](https://opendart.fss.or.kr)에서 무료 발급). 프로젝트 루트에 `.env` 파일을 만들고 `.env.example`을 참고해 아래와 같이 작성하세요.

```
DART_API_KEY=발급받은_키
```

`.env`는 `.gitignore`에 포함되어 있어 저장소에 올라가지 않습니다.

## 사용법

```bash
python main.py "https://dart.fss.or.kr/dsaf001/main.do?rcpNo=20260323001196&dcmNo=11166205"

# 별도재무제표 기준으로 강제 조회 (기본값은 아래 자동 판별)
python main.py "<DART 링크>" --fs-div OFS

# 결과 저장 위치 지정
python main.py "<DART 링크>" --output-dir ./output
```

`--fs-div`를 지정하지 않으면 다음 순서로 연결/별도를 자동 판별합니다: (1) 링크에 `dcmNo`가 있으면 그 문서가 "감사보고서"(별도)인지 "연결감사보고서"(연결)인지 확인해 우선 사용 — 사용자가 실제로 클릭해서 가져온 링크의 의도를 최우선으로 반영, (2) dcmNo가 없거나 판별 불가면 연결재무제표(CFS)를 먼저 시도, (3) 그 기준의 재무제표가 실제로 공시되어 있지 않으면 반대쪽으로 자동 전환. 계정과목 내용(비지배지분 유무 등)으로 추론하지 않고 링크·API 응답 자체로만 판별합니다. 실행하면 콘솔에 어떤 근거로 연결/별도가 선택됐는지 안내가 출력되고, `{회사명}_{연결|별도}_결과.xlsx` 파일이 생성됩니다(안내/재무제표/재무비율/상세보기_원항목추이/스크리닝_전기당기증감/검증 6개 시트 + 그래프 포함, "안내" 시트에 판단 근거까지 명시).

## 적용 범위 / 한계

본 도구는 제조업/서비스업 등 일반 기업의 표준 재무비율 분석에 최적화되어 있습니다. 금융업(은행/보험/증권/지주회사)은 재무제표 구조가 달라 정확한 분석이 어려우며, 현재 지원 범위에서 제외되어 있습니다. 금융업종 분석 기능은 향후 추가를 검토 중입니다.

- 은행/보험/증권/금융지주는 유동/비유동 구분이 없는 유동성배열법 재무상태표를 쓰고 매출액 대신 이자수익·보험수익·수수료수익 등을 핵심 손익으로 인식하기 때문에, 유동비율·매출총이익률·영업이익률·순이익률 등이 계정 매핑 실패로 공란 처리됩니다(계산 오류가 아니라 업종 특성). 실제 삼성화재(보험)/우리금융지주(금융지주)/기업은행(은행)/삼성증권(증권사) 4개 업종으로 검증했습니다.
- 계정과목명(예: "이자수익", "보험계약부채", "예수부채")으로 금융업 여부를 자동 감지해(`src/industry_detect.py`) 콘솔과 엑셀 "안내" 시트에 눈에 띄는 경고를 표시합니다. DART 업종코드(induty_code)는 순수 지주회사와 금융지주회사를 구분하지 못해(둘 다 64992로 동일) 보조 신호로 채택하지 않았습니다.

## 면책 사항

본 도구의 산출물은 감사 착수 전 참고용 사전분석 자료이며, 감사의견 형성의 유일한 근거가 될 수 없습니다.
